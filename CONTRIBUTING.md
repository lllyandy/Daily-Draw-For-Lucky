# 贡献指南

感谢您对观音灵签项目的关注！

## 如何贡献

### 报告问题
如果您发现了 bug 或有改进建议，请提交 Issue：
1. 描述问题
2. 说明复现步骤
3. 提供环境信息（Node.js 版本、操作系统等）

### 提交代码
1. Fork 本仓库
2. 创建您的特性分支 (`git checkout -b feature/AmazingFeature`)
3. 提交您的更改 (`git commit -m 'Add some AmazingFeature'`)
4. 推送到分支 (`git push origin feature/AmazingFeature`)
5. 打开一个 Pull Request

### 签文改进
如果您对传统签文有更好的解释或典故补充，欢迎提交 PR。

### 代码规范
- 使用清晰的变量名
- 添加必要的注释
- 保持代码简洁

## 行为准则

- 尊重他人
- 接受建设性批评
- 关注社区最佳利益
